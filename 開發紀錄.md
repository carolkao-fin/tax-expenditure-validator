# 稅式支出評估報告驗證工具 — 開發紀錄

## 專案概述

**目的**：讓財政部或研究人員上傳 Word 格式的「稅式支出評估報告」，自動驗證計算是否正確，並匯出 Excel 驗證報告。  
**部署**：Streamlit Cloud，GitHub repo `carolkao-fin/tax-expenditure-validator`  
**技術棧**：Python、Streamlit、python-docx、openpyxl

---

## 支援文件格式

| 格式 | 單位 | 特徵 |
|------|------|------|
| 完整版（如「汽車零組件關稅調降稅式支出評估報告」.F2.docx） | 億元 | 13 個 HS 稅則品項、表4-1/4-2、各稅目公式計算表（表5~12）|
| 簡要格式（如「稅式支出評估報告(簡要格式)」.F2.docx） | 千元 | 一張 61 列大表、末端有 8 列彙總區塊 |

---

## 功能說明

### 驗證項目（完整版）
- 5年平均進口總額加總（表4-1）
- 算術平均現行稅率
- 各 HS 品項關稅損失（進口值 × 降稅幅度）
- 關稅損失合計（表4-2）
- 貨物稅（整車產值 × 各比例 × 稅率）
- 加值型營業稅
- 整車 / 零組件 / 其他工業及服務業 CIT
- 個人綜合所得稅（員工數 × 平均薪資 × 適用稅率）
- 股東個人股利所得稅（利潤 × (1-CIT) × 分配率 × 個人稅率）
- 最終收入損失法淨損益

### 驗證項目（簡要格式）
- 5年平均進口總額
- 關稅損失
- 貨物稅（整車產值基數）
- 加值型營業稅
- 整車 / 零組件 / 其他工業及服務業 CIT（千元）
- 個人綜合所得稅
- 股東個人股利所得稅
- 最終收入損失法淨損益
- 附：表格數值核對（對照彙總表數字）

### 輸出
- 網頁即時顯示（按章節分組，綠/紅/黃色列）
- 每份檔案各自下載 Excel，檔名 `驗證結果_XXXX.xlsx`
- Excel 含：章節、驗證項目、公式原文、報告值、計算值、差異、結果、說明

### 62e557c — 表格頁碼追蹤根本修正

**問題**：整張跨頁表格只記錄起始頁碼，61 列簡表全部顯示「第1頁」。  
**錯誤嘗試**：在換頁點切割表格為多個 chunk → `find_table_by_header` 只找到含標題的 chunk（可能只有 1 列），資料列全部消失。

**正確解法**：
- `table_list` 每個 entry 改為 3-tuple：`(start_page, rows, row_pages)`
- `row_pages[i]` = 第 i 列的實際頁碼（在迭代每列時，遇 `w:lastRenderedPageBreak` 才遞增頁碼）
- 完整表格仍作為一個 entry，`find_table_by_header` 依然看到所有列
- `scan_tables_financial` 和 `parse_simplified_summary_table` 記錄 `(值, row_page)` 而非整張表的起始頁
- `build_paged_text` 也改為用每列自己的頁碼放入 `by_page` dict
- 新增 `_tbl(entry)` helper 解包（向後相容 2-tuple 格式）

### 77cc5cf / f067935 / 4ae4f8b — 重構：移除 PDF 支援與頁碼功能，改用 python-docx Table API

**移除**：
- 移除 pdfplumber 依賴（`requirements.txt` 更新），不再支援 PDF 上傳
- 移除頁碼追蹤機制（`w:lastRenderedPageBreak` 掃描）及 Excel「資料頁碼」欄，簡化文件模型

**原因**：頁碼機制複雜且維護成本高；完整版文件以 Word 格式提交，PDF 支援為次要需求。

**改用 python-docx Table API**：
- 原本直接掃 XML 取表格內容；改用 `doc.tables` → `table.rows` → `row.cells` 路徑
- 合併儲存格（rowspan/colspan）會讓同一 `<w:tc>` 出現多次 → 用 `id(cell._tc)` 去重
- 表格 entry 格式退回 2-tuple：`(title: str, rows: [[str]])`

### 5728c09 — 滾動段落緩衝區（rolling buffer）+ SDT wrapper 支援

**核心問題**：表格標題（如「表4-1 近5年…」）出現在表格前 2–4 個段落，舊版只抓緊鄰上一段 → 公式表格的 `title` 幾乎為空，無法以關鍵字識別。

**解法**：`extract_document()` 改用 `para_buf`（WINDOW=4）記錄最近 4 個非空段落，每張表格的 `title = ' | '.join(para_buf)`，之後 reset。

```python
WINDOW = 4
para_buf = []
# ...
context = ' | '.join(para_buf)   # 最多 4 個前置段落
table_list.append((context, rows))
para_buf.clear()
```

**額外修正**：`_visit()` 新增對 `w:sdt`（Content Control wrapper）的遞迴下探，確保被 SDT 包裹的段落不被略過。

### 5006034 — 表4-1 結構偵測優先 + 公式表格依前置段落識別

**問題**：`find_table_by_header(table_list, '現行稅率', '降稅後稅率')` 按文件順序搜尋，先找到「表格1」（9 個 HS 行）而非「表格3」（13 個 HS 行，即表4-1） → 對錯表。

**修正**：`parse_full_report()` 改以 `find_hs_table()` 作為第一優先：HS 碼行數最多者即為表4-1，關鍵字策略退為 fallback。

**公式表格識別改進**：`has(kw)` 原本只掃 `cell_text`；改掃 `combined = title + '\n' + cell_text`，使前置段落的識別關鍵字（如「汽車整車」）也能被偵測到。

### 7a71087 — 表4-2 結構偵測優先 + 公式表格尺寸 × 運算子雙重門檻

**表4-2**：同理以 `find_hs_table(table_list, skip=tbl41)` 為第一優先，跳過已找到的 tbl41 → 取 HS 碼行數次多的表格。

**公式表格誤判問題**：進口資料表（15+ 行）若含關鍵字（如服務業），會被錯誤歸入公式表格。

**修正（雙重門檻）**：
```python
if not rows or len(rows) > 3 or '×' not in '\n'.join(' '.join(r) for r in rows):
    continue
```
- `len(rows) > 3`：公式表格通常 1–3 列，進口資料表 15+ 列 → 直接過濾
- `'×' not in cell_text`：公式表格必有乘號 → 排除股東所得稅級距表（10 列、無 ×）

### 7a487aa — 股利/個人所得稅 elif 排序修正 + 段落文字搜尋

**問題 1：elif 排序錯誤**  
`has('個人所得稅')` 是 `has('股東個人股利所得稅')` 的子字串；原本 `personal` 的 elif 排在 `dividend` 之前 → 股利公式表格被歸入 personal。

**修正**：`dividend` 的 elif 移至 `personal` 之前；收緊 personal 條件（必須含員工關鍵字，明確排除股利/股東）：
```python
elif 'dividend' not in formula_tables and (has('股利') or (has('股東') and has('盈餘'))):
    formula_tables['dividend'] = rows

elif 'personal' not in formula_tables and (
        has('增聘') or has('受雇員工') or
        (has('薪資') and not has('股利') and not has('股東')) or ...):
    formula_tables['personal'] = rows
```

**問題 2：公式表格值不穩定**  
股利（Table 10）與個人所得稅（Table 11）儲存格各含多段子計算 + 總值，從表格抽取時易取到中間子計算值。

**修正**：改以段落文字搜尋作為第一優先，直接找含明確金額聲明的段落：
```python
# 股利：找「股東個人股利所得稅增加新臺幣 X 億元」
m = re.search(r'股(?:東個人股利|東股利|利)[^。\n]{0,200}新臺幣\s*([\d\.]+)\s*億元',
              all_paras, re.DOTALL)

# 個人：找「員工個人所得稅增加新臺幣 X 萬元」
m = re.search(r'(?:員工個人所得稅|個人綜合所得稅)[^。\n]{0,200}新臺幣\s*([\d,]+)\s*萬元',
              all_paras)
```

### b19e8b1 — 公式總值擷取三項修正

**修正 1：`compact_formula_full()` 取最後結果行**  
股利公式儲存格有多行 `新臺幣 X 億元`（子計算 0.11、0.59 … 總值 0.77）；`next()` 取第一行 → 顯示錯誤子計算值。

```python
# reversed() 取最後一行，即公式最終總值
result_line = next((l for l in reversed(lines) if '新臺幣' in l and '億元' in l), None)
```

**修正 2：個人所得稅萬元 regex 容許「新臺幣」前綴**  
文件格式：`= 新臺幣287.4萬元`；原 regex `=\s*([\d\.]+)\s*萬元` 不匹配（數字前有「新臺幣」）→ 只捕捉到子計算 37.4 萬元。

```python
vals = re.findall(r'=\s*(?:新臺幣)?\s*([\d\.]+)\s*萬元', txt)
per_formula_val = float(vals[-1]) / 10000   # 取最後一個值（總額 287.4）
```

**修正 3：個人所得稅段落搜尋關鍵字**  
文件段落標題為「員工個人所得稅」而非「個人綜合所得稅」→ 原 regex 無匹配。

```python
r'(?:員工個人所得稅|個人綜合所得稅)[^。\n]{0,200}新臺幣\s*([\d\.]+)\s*億元'
```

## 關鍵技術說明

### 滾動段落緩衝區（Rolling Buffer）

表格標題段落通常出現在表格前 2–4 個段落，而非緊鄰前一段。`extract_document()` 維護一個長度為 4 的滾動緩衝區，每張表格的識別標題 = 最近 4 個段落的拼接；表格後立即清空緩衝區，確保下一張表格的標題不受污染。

### HS 碼結構偵測

```python
def _count_hs_rows(rows):
    return sum(1 for r in rows if r and re.match(r'^\d{4}', r[0].strip()))

def find_hs_table(table_list, skip=None):
    best_rows, best_n = None, 1
    for title, rows in table_list:
        if rows is skip: continue
        n = _count_hs_rows(rows)
        if n > best_n:
            best_n, best_rows = n, rows
    return best_rows
```

第一欄符合 `^\d{4}` 格式的列數最多者 = 進口資料表。跳過已識別的 tbl41 取次多者 = tbl42。

### 公式表格雙重門檻

```python
if not rows or len(rows) > 3 or '×' not in '\n'.join(' '.join(r) for r in rows):
    continue
```

進口資料表：15+ 行、無 × → 被 `len(rows) > 3` 過濾。  
股東所得稅級距表：10 行、無 × → 被 `'×' not in cell_text` 過濾。  
公式計算表：1–3 行、有 × → 通過。

### 簡要格式彙總表偵測

```
尋找任何 table entry 中含「最初收入損失法」+「最終收入損失法」的標題列
→ 記下「最終收入損失法」欄的欄位索引
→ 逐列讀縮寫標籤（關稅/貨物稅/營業稅/合計...）對應 key
→ 取 abs() 處理負值（關稅 = -2,240,360）
```

### 表格數值掃描三種 Layout

| Layout | 條件 | 取值方式 |
|--------|------|---------|
| A | 千元/億元直接寫在格子裡 | regex `數字+單位` |
| B | 單位只在欄位標題 | 找到標題欄 index，讀對應欄位值 |
| C | 沒有單位，但有稅目關鍵字 | 最右側純數字格，門檻 ≥ 1000 |
