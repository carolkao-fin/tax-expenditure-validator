# 稅式支出評估報告驗證工具 — 開發紀錄

## 專案概述

**目的**：讓財政部或研究人員上傳 Word 格式的「稅式支出評估報告」，自動驗證計算是否正確，並匯出 Excel 驗證報告。  
**部署**：Streamlit Cloud，GitHub repo `carolkao-fin/tax-expenditure-validator`  
**技術棧**：Python、Streamlit、python-docx、openpyxl、pdfplumber

---

## 支援文件格式

| 格式 | 單位 | 特徵 |
|------|------|------|
| 完整版（如「汽車零組件關稅調降稅式支出評估報告」.F2.docx） | 億元 | 13 個 HS 稅則品項、表4-1/4-2、各稅目公式計算表（表5~12）|
| 簡要格式（如「稅式支出評估報告(簡要格式)」.F2.docx） | 千元 | 一張 61 列大表、末端有 8 列彙總區塊 |

---

## 功能說明

### 驗證項目（完整版）
- 5年平均進口總額加總（表4-1）
- 算術平均現行稅率
- 各 HS 品項關稅損失（進口值 × 降稅幅度）
- 關稅損失合計（表4-2）
- 貨物稅（整車產值 × 各比例 × 稅率）
- 加值型營業稅
- 整車 / 零組件 / 其他工業及服務業 CIT
- 個人綜合所得稅（員工數 × 平均薪資 × 適用稅率）
- 股東個人股利所得稅（利潤 × (1-CIT) × 分配率 × 個人稅率）
- 最終收入損失法淨損益

### 驗證項目（簡要格式）
- 5年平均進口總額
- 關稅損失
- 貨物稅（整車產值基數）
- 加值型營業稅
- 整車 / 零組件 / 其他工業及服務業 CIT（千元）
- 個人綜合所得稅
- 股東個人股利所得稅
- 最終收入損失法淨損益
- 附：表格數值核對（對照彙總表數字）

### 輸出
- 網頁即時顯示（按章節分組，綠/紅/黃色列）
- 每份檔案各自下載 Excel，檔名 `驗證結果_XXXX.xlsx`
- Excel 含：章節、驗證項目、公式原文、報告值、計算值、差異、結果、說明、**資料頁碼**

Excel 驗證報告新增「資料頁碼」欄（如「第3頁」），追蹤每項驗證數值的來源頁碼。

**頁碼追蹤機制（Word）**：
- 掃描 XML 中的 `w:lastRenderedPageBreak`（Word 自動換頁標記）與 `w:br w:type="page"`（手動換頁）
- 段落：換頁記號在文字前 → 先加頁、再記錄；在文字後 → 記錄後加頁
- `build_paged_text()` 將所有內容按頁拼接，`get_page(match_start)` 依字元位置回傳頁碼

### f170998 — 頁碼與表格偵測修正
- 修正段落換頁計算邏輯，避免在同一段落多次計入
- `find_table_by_header()` 改為搜尋前 3 列（處理有合併標題列的表格）

### cdff5e8 — 簡要格式彙總表解析
**問題根因**：簡要格式文件末端有 8 列彙總區塊，使用縮寫標籤（`關稅`、`營業稅`、`合計`），與 `_KEYWORD_MAP` 的完整名稱不符；且 Layout C 的門檻 ≥1000 擋掉了 2,874（個人綜合所得稅）。

**修正**：新增 `parse_simplified_summary_table()`，直接尋找含「最初收入損失法」+「最終收入損失法」標題列的區塊，使用簡化對照表解析縮寫，取最終損益法欄位值，負值取絕對值。

同時修正 `rep_net` 的 fallback 預設值 29,231 → 29,241（文件正確值）。

### 4d4dc99 — PDF 上傳支援
- 新增 `extract_pdf()` 使用 pdfplumber，PDF 每頁即為獨立邊界，頁碼精確
- 表格用 `page.find_tables()` 抽取，文字部分用 `page.filter()` 排除表格區域，避免重複

### 3b34cf4 — 完整版偵測強化 + 個人/股利所得稅計算
- `find_table_by_header()` 已改掃前 3 列
- 表4-2 新增四種備用偵測策略（關鍵字組合、HS 碼列數量啟發式）
- 個人/股利所得稅從「依文件確認」升級為嘗試自動計算：
  - 個人所得稅：抽取員工數、平均年薪、適用稅率 → 三項齊全才計算
  - 股利所得稅：抽取利潤基數、分配率、個人稅率 → 齊全才計算

### 62e557c — 表格頁碼追蹤根本修正

**問題**：整張跨頁表格只記錄起始頁碼，61 列簡表全部顯示「第1頁」。  
**錯誤嘗試**：在換頁點切割表格為多個 chunk → `find_table_by_header` 只找到含標題的 chunk（可能只有 1 列），資料列全部消失。

**正確解法**：
- `table_list` 每個 entry 改為 3-tuple：`(start_page, rows, row_pages)`
- `row_pages[i]` = 第 i 列的實際頁碼（在迭代每列時，遇 `w:lastRenderedPageBreak` 才遞增頁碼）
- 完整表格仍作為一個 entry，`find_table_by_header` 依然看到所有列
- `scan_tables_financial` 和 `parse_simplified_summary_table` 記錄 `(值, row_page)` 而非整張表的起始頁
- `build_paged_text` 也改為用每列自己的頁碼放入 `by_page` dict
- 新增 `_tbl(entry)` helper 解包（向後相容 2-tuple 格式）

## 關鍵技術說明

### Word XML 頁碼追蹤

```python
# w:lastRenderedPageBreak 出現在 w:r 子元素中，位於文字之前代表該段/列在新頁開始
for child in run:
    cl = child.tag.split('}')[-1]
    if cl == 'lastRenderedPageBreak' and not seen_text:
        breaks_before += 1
    elif cl == 'br' and child.get(qn('w:type')) == 'page' and not seen_text:
        breaks_before += 1
    elif cl == 't' and child.text.strip():
        seen_text = True
current_page += breaks_before
```

### 簡要格式彙總表偵測

```
尋找任何 table entry 中含「最初收入損失法」+「最終收入損失法」的標題列
→ 記下「最終收入損失法」欄的欄位索引
→ 逐列讀縮寫標籤（關稅/貨物稅/營業稅/合計...）對應 key
→ 取 abs() 處理負值（關稅 = -2,240,360）
→ 使用該列自己的 row_page 作為頁碼
```

### 表格數值掃描三種 Layout

| Layout | 條件 | 取值方式 |
|--------|------|---------|
| A | 千元/億元直接寫在格子裡 | regex `數字+單位` |
| B | 單位只在欄位標題 | 找到標題欄 index，讀對應欄位值 |
| C | 沒有單位，但有稅目關鍵字 | 最右側純數字格，門檻 ≥ 1000 |
